# SHORE Environment Variables

## Web

| Variable                   | Local example           | Purpose                            |
| -------------------------- | ----------------------- | ---------------------------------- |
| `APP_ENV`                  | `local`                 | Runtime environment label          |
| `API_BASE_URL`             | `http://localhost:8787` | Server-side API origin             |
| `NEXT_PUBLIC_API_BASE_URL` | `http://localhost:8787` | Browser API origin when required   |
| `NEXT_PUBLIC_TON_NETWORK`  | `testnet`               | Wallet network shown to the client |

## API

| Variable      | Local example           | Purpose                            |
| ------------- | ----------------------- | ---------------------------------- |
| `APP_ENV`     | `local`                 | `local`, `staging` or `production` |
| `CORS_ORIGIN` | `http://localhost:3000` | Allowed Web origin                 |
| `LOG_LEVEL`   | `debug`                 | Structured log threshold           |

Future secrets such as session signing keys, Telegram bot tokens and AI provider keys must be configured with Cloudflare Secrets. They do not belong in Wrangler `vars`, local source files or Git history.

## Bindings

The API expects these Cloudflare bindings:

```text
DB            D1Database
PRIVATE_FILES R2Bucket
ASYNC_JOBS    Queue
```

The Web application expects:

```text
ASSETS        Workers Assets binding generated by OpenNext
```

## Local secret file

For local API development, create `apps/api/.dev.vars` manually. This path is ignored by Git.

Do not copy production secrets into local development. Use dedicated local or Staging credentials.
