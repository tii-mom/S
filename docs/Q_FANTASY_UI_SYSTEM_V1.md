# SHORE 单页十八轮 Q 版交互系统 V3

## 1. 页面模型

前端仅保留一个纵向业务页面：

```text
/
```

页面由一条连续弯曲路线和十八个圆形关卡节点组成，对应十八轮解锁。

页面不提供：

- 顶部导航；
- 底部导航；
- 多级菜单；
- 其他业务页面；
- 人物、角色或宠物；
- 长篇产品介绍。

所有功能都通过地图上的关卡、两个状态球、底部唯一操作按钮和底部操作面板完成。

## 2. 信息层级

单页面分为三层。

### 第一层：十八轮地图

负责表达：

- 当前所处轮次；
- 已完成轮次；
- 未解锁轮次；
- 当前轮进度；
- 下一轮位置。

### 第二层：轻量状态组件

页面顶部只保留两个固定 Q 版状态球：

- 左侧：上岸行动进度；
- 右侧：SHORE 可用数量与待领取提示。

状态球不是导航栏，点击后只打开对应底部面板。

### 第三层：当前操作

页面底部只保留一个主要按钮。按钮根据用户状态自动变化：

```text
开始上岸
开始任务
提交证明
连接钱包
领取 SHORE
等待下一轮
```

用户不需要判断应该进入哪个页面。

## 3. 文字原则

V3 不再执行“一个字都不出现”。

正确比例为：

```text
90% 视觉组件 + 10% 必要短文字
```

允许显示：

- 轮次数字；
- 百分比；
- 金额；
- 积分与 SHORE 数量；
- 上传、开始、提交、连接、领取、完成等操作词；
- 底部面板中的简短任务步骤和状态原因。

禁止显示：

- 大段宣传文案；
- 复杂规则说明；
- 多栏菜单；
- 白皮书式内容；
- 诱导购买或承诺收益的文字。

## 4. 用户闭环

当前 Mock 交互路径：

```text
点击“开始上岸”
→ 输入负债总额或使用账单识别示例
→ 点击“开始任务”
→ 查看三步今日任务
→ 点击“提交证明”
→ 粘贴链接或添加 Mock 截图
→ 获得 300 积分
→ 连接 TON 钱包 Mock
→ 领取 150,000 SHORE Mock
→ 第 4 轮完成
→ 第 5 轮成为当前轮
```

全部操作在同一页面的底部面板中完成。

## 5. 关卡状态

十八个节点按照三种状态展示：

### 已完成

- 彩色节点；
- 绿色勾选；
- 完整进度环；
- 已点亮路线；
- 点击后查看历史结果。

### 当前轮

- 彩色高亮；
- 呼吸动画；
- 外圈进度；
- 用户完成条件后进入可领取光效；
- 点击后查看价格、行动与个人状态。

### 未解锁

- 低饱和紫色；
- 锁定图标；
- 点击后查看还未满足的解锁条件；
- 不允许执行领取。

轮次数字 1–18 直接显示在节点下方，帮助用户理解路线顺序。

## 6. 底部操作面板

底部面板承载以下模式：

1. 设置或修改负债目标；
2. 查看行动进度；
3. 查看 SHORE 资产；
4. 查看任意轮次状态；
5. 查看今日任务；
6. 提交 Proof of Action；
7. 连接 TON 钱包 Mock；
8. 领取 SHORE Mock；
9. 查看验证或领取成功状态。

面板规则：

- 每次只显示一个主要按钮；
- 关闭后回到原地图位置；
- 面板打开时锁定背景滚动；
- 支持手机安全区；
- 不进入新路由；
- 错误和限制使用明确短文案说明。

## 7. 状态与持久化

当前使用版本化浏览器本地状态：

```text
shore.experience.v3
```

保存：

- 负债目标；
- 当前任务状态；
- Proof 链接；
- 积分；
- 可用 SHORE Mock；
- 钱包 Mock 连接状态；
- 第 4 轮领取状态；
- 分享次数。

状态机必须防止：

- 同一个任务重复获得积分；
- 同一轮重复领取；
- 未完成任务直接领取；
- 未连接钱包直接领取。

真实账户系统上线后，由服务端会话、D1 账本和链上索引替换本地状态。

## 8. Q 版视觉语言

视觉参考上传图片中的通用结构语言：

- 深蓝紫色背景；
- 自上而下的成长路线；
- 浅色宽轨道；
- 交替分布的关卡节点；
- 漂浮的小型装饰组件；
- 终点奖励物；
- 粗描边、圆润、高光和轻量弹性动画。

不得复制参考图中的人物、品牌、Logo、文字、图标或具体美术资源。

允许使用的原创组件：

- 宝石、星星、钥匙、盾牌；
- 月亮、太阳、药水瓶、卷轴；
- 贝壳、四叶草、勋章、戒指；
- 皇冠、礼盒、宝箱、城堡；
- 云朵、花朵和光点。

## 10. Mock 边界

当前交互必须明确：

- 负债数据仅保存在浏览器本地；
- 上传账单不会发送真实文件；
- AI 识别是示例；
- Proof 审核是即时 Mock；
- TON Connect 不连接真实钱包；
- 领取不发起链上交易；
- SHORE 数量只是交互演示；
- 不展示真实收益承诺。

## 11. 响应式规则

- 页面最小宽度为 320px；
- 地图画布最大宽度为 720px；
- 手机端保持足够纵向关卡间距；
- 固定状态球与主按钮不能遮挡系统安全区；
- 所有关卡、状态球和操作按钮必须位于视口宽度以内；
- 页面允许纵向滚动，不允许横向滚动；
- 底部面板最大高度不超过视口的 84%。

## 12. 自动验收

E2E 必须在以下视口执行：

```text
375 × 812
430 × 932
1280 × 900
```

必须验证：

- 只有一个业务页面；
- 恰好十八个关卡节点；
- 没有 header 和 nav；
- 顶部只有两个状态球；
- 底部只有一个主要操作；
- 锁定关卡可以查看条件；
- 完整完成目标、任务、证明、钱包与领取路径；
- 领取后第 4 轮变为完成、第 5 轮变为当前；
- 防止重复积分和重复领取；
- 固定组件和节点不超出视口；
- 页面无横向溢出；
- 浏览器控制台无错误。
