# 《上岸》产品开发文档 V1

- 产品名称：上岸
- 英文名称：SHORE
- 代币名称：上岸币
- 代币符号：SHORE
- 文档版本：V1
- 产品阶段：MVP / 受控测试设计

---

## 一、产品定义

### 1. 一句话介绍

> 上传负债，完成任务，领取上岸币。

### 2. 首页核心口号

> 负债不是身份，上岸才是目标。

### 3. 用户三秒理解路径

```text
上传负债 → 完成任务 → 领取上岸币
```

用户只需要理解四件事：

1. 我还欠多少钱；
2. 今天需要完成什么；
3. 完成后能得到什么；
4. 我距离上岸还有多远。

### 4. 最终定位

> 上岸是一款让负债用户通过完成真实任务，积累奖励和上岸币，逐步靠近负债目标的傻瓜式行动应用。

### 5. 最终口号

> 每天做一件事，离上岸近一步。

---

## 二、产品核心原则

### 1. 一个页面只有一个主要操作

- 首页：开始今天的上岸任务；
- 任务页：去完成；
- 提交页：提交证明；
- 金库页：领取。

### 2. 第一次使用不要求钱包

用户不需要先完成以下操作：

- 创建钱包；
- 购买 TON；
- 理解区块链；
- 阅读白皮书；
- 授权全部社交账号；
- 完成复杂身份认证。

用户先使用产品、建立负债目标、完成任务并获得积分。只有需要领取 SHORE 时，才引导连接 TON 钱包。

### 3. 日常操作保持固定

```text
打开产品 → 查看今日任务 → 完成任务 → 提交链接或截图 → 获得奖励
```

### 4. V1 明确不开发

- 小队系统；
- 用户等级；
- 复杂信誉体系；
- NFT；
- SBT；
- DAO 治理；
- 多链；
- 聊天系统；
- 自动理财；
- 自动交易；
- 复杂债务咨询；
- 广告主自助后台；
- 复杂排行榜；
- 自动代替用户发布、点赞、评论或私信。

## 三、核心产品闭环

### 1. 用户闭环

```text
用户上传负债
    ↓
AI 计算上岸目标
    ↓
用户每天完成一个任务
    ↓
用户获得积分和代币权益
    ↓
用户分享上岸进度
    ↓
带来更多用户
    ↓
更多用户完成平台任务
    ↓
平台获得更多曝光与商业客户
    ↓
广告主购买任务
    ↓
平台收入支持用户奖励与代币需求
```

### 2. 代币价值闭环

```text
广告主支付任务预算
    ↓
用户完成推广、测试、内容或反馈任务
    ↓
广告主获得真实曝光、用户或反馈
    ↓
平台收取服务费
    ↓
部分服务费回购 SHORE
    ↓
用户持续持有、使用和领取 SHORE
```

代币价值不能只依赖用户转发。核心是用户完成能够为广告主或平台创造真实结果的任务。

## 四、第一次使用流程

### 第一步：进入产品

支持：

- Telegram Mini App；
- 独立 Web；
- PWA；
- Telegram 自动登录；
- 邮箱登录。

首次进入不要求钱包。

### 第二步：输入负债

页面文案：

> 你现在总共欠多少钱？

两个主要入口：

- 上传负债截图；
- 直接输入金额。

用户可以上传：

- 信用卡账单；
- 银行贷款截图；
- 网贷截图；
- 房贷、车贷截图；
- 其他欠款记录；
- PDF；
- 多张图片。

### 第三步：AI 计算

AI 只完成三件事：

1. 识别金额；
2. 去除明显重复；
3. 计算总额。

结果示例：

```text
AI 识别负债总额
¥186,420

识别了 7 笔负债
```

操作：

- 确认金额；
- 修改金额。

详细负债明细默认折叠，用户无需逐项填写机构、利率或还款日期。

### 第四步：生成上岸目标

```text
你的上岸目标

¥186,420

从今天开始，每完成一个任务，
你就距离上岸更近一步。
```

主按钮：

> 开始第一个任务

## 六、首页

### 1. 首页目标

用户进入后立即知道：

- 还欠多少钱；
- 已经获得多少；
- 有多少待解锁；
- 今天需要完成什么。

### 2. 页面结构

#### 上岸进度

最大数字：

```text
距离上岸还差
¥176,850
```

辅助数据：

```text
原始负债      ¥186,420
已经获得        ¥9,570
上岸进度          5.13%
```

“已经获得”可包含：

- 稳定价值任务奖励；
- 可领取代币的当前参考价值；
- 用户手动记录的真实还款金额。

锁仓代币不能计入“已经获得”，只能显示在待解锁资产中。

#### 今日上岸任务

首页只推荐一个任务：

```text
今日上岸任务

在 X 发布一条原创内容
预计时间：3 分钟
奖励：300 积分
```

主按钮：

> 去完成

#### 我的上岸币

只显示三个数字：

```text
可用 SHORE        100,000
待解锁 SHORE    2,700,000
当前轮次            第 0 轮
```

按钮：

> 查看金库

#### 全网数据

只提供简单社会证明：

```text
今日完成任务       18,520 项

本周新增任务           126 个
```

#### 分享入口

主文案：

## 七、负债功能

### 1. 上传方式

- 单张图片；
- 多张图片；
- PDF；
- 手动输入总额。

### 2. 数据结构

```ts
type DebtSummary = {
  id: string;
  userId: string;
  currency: string;
  aiDetectedAmount: number;
  confirmedAmount: number;
  itemCount: number;
  status: "processing" | "waiting_confirmation" | "confirmed";
  createdAt: string;
  updatedAt: string;
};
```

### 3. AI 输出格式

```json
{
  "currency": "CNY",
  "detectedTotal": 186420,
  "itemCount": 7,
  "confidence": 0.91,
  "suspectedDuplicates": 1
}
```

### 4. 隐私要求

- 自动识别并遮挡姓名；
- 自动识别并遮挡身份证号；
- 自动识别并遮挡银行卡号；
- 原图使用私有存储；
- 用户可以删除原图；
- 管理员默认无法浏览用户完整原图；
- 分享卡默认不显示债权人；
- 分享卡默认只显示上岸百分比。

分享时提供两个选项：

- 只显示百分比，默认；
- 显示具体金额，用户主动选择。

### 5. V1 不提供

- 还款策略；
- 利率分析；
- 债务重组建议；
- 贷款推荐；
- 信用评分；
- 投资建议。

## 八、任务系统

### 1. 页面原则

任务页默认展示：

- 一个今日推荐任务；
- 最多三个可选任务。

同时展示的任务不超过四个，避免用户选择困难。

### 2. V1 任务类型

#### 社交发布

- 发布原创帖子；
- 发布短视频；
- 发布产品体验；
- 创建相关话题；
- 分享活动。

#### 社交互动

- 加入 Telegram 频道；
- 参加群组活动；
- 回复官方问题；
- 参加 AMA；
- 完成公开投票。

禁止奖励无意义批量刷屏、批量点赞或虚假评论。

#### 产品体验

- 启动 Telegram Bot；
- 打开 Mini App；
- 完成新手流程；
- 测试一个功能；
- 提交体验反馈。

#### 内容创作

- 制作图片；
- 制作 Meme；
- 录制短视频；
- 编写教程；
- 翻译内容。

#### 上岸分享

- 分享个人上岸进度；
- 邀请一位有效新用户；
- 发布个人行动记录。

### 3. 任务详情页

每个任务只显示：

- 任务标题；
- 任务奖励；
- 预计时间；
- 最多三个操作步骤；
- 提交要求。

示例：

```text
发布一条上岸主题原创内容

奖励：300 积分
预计时间：3 分钟

1. 查看内容要求
2. 在 X 发布原创内容
3. 返回并提交帖子链接
```

主按钮：

> 去完成

### 4. 提交方式

用户只需要选择一种：

- 粘贴链接；
- 上传截图。

支持 Webhook 或 Bot 回传时，不要求用户重复提交。

### 5. 验证流程

```text
用户提交
    ↓
检查链接是否有效
    ↓
AI 检查内容是否满足要求
    ↓
检查是否重复
    ↓
通过或进入人工审核
```

成功文案：

```text
验证成功
奖励已到账
```

失败文案必须给出具体原因：

```text
需要重新提交
原因：链接无法打开
```

### 6. 任务数据结构

```ts

  id: string;
  title: string;
  description: string;
  platform: string;
  rewardPoints: number;
  rewardToken?: string;

  completedParticipants: number;
  verificationType: "url" | "image" | "webhook";
  status: "draft" | "active" | "paused" | "completed";
  startsAt: string;
  endsAt: string;
};
```

```ts
type TaskSubmission = {
  id: string;
  userId: string;
  taskId: string;

  proofImageKey?: string;
  status:
    | "pending"
    | "ai_review"
    | "manual_review"
    | "approved"
```

### 7. AI 审核输出

```json
{
  "result": "approved",

  "duplicateRisk": 0.08,
```

## 十、积分系统

### 1. 名称

统一使用“积分”，不创建其他复杂名称。

### 2. 三个用途

1. 兑换每轮社区代币；
2. 满足创世用户领取条件；
3. 获得更多高奖励任务。

### 3. 积分示例

| 任务 | 积分 |
|---|---:|
| 绑定一个社交账号 | 100 |
| 完成产品测试 | 200 |
| 发布原创内容 | 300 |
| 制作短视频 | 800 |
| 邀请有效用户 | 500 |

### 4. 有效邀请条件

新用户必须完成：

1. 确认负债目标；
2. 绑定一个社交账号；
3. 完成第一个任务。

全部完成后，邀请双方获得积分。

### 5. 规则

- 积分不可转账；
- 积分不可购买；
- 积分不可直接提现；
- 作弊积分可以撤销；
- 所有积分变化必须记录到账本；
- 管理员不得无日志修改积分余额。

## 十一、代币经济

### 1. 总供应量

```text
10,000,000,000 SHORE
```

总量固定为 100 亿，禁止增发。

### 2. 初始解锁

合约部署后解锁：

```text
1,000,000,000 SHORE
```

占总供应量 10%。

### 3. 后续解锁

剩余：

```text
9,000,000,000 SHORE
```

分 18 轮解锁，每轮：

```text
500,000,000 SHORE
```

### 4. 每轮分配

| 用途 | 比例 | 每轮数量 |
|---|---:|---:|
| 创世购买者 | 40% | 200,000,000 |
| 积分兑换 | 40% | 200,000,000 |
| 团队 | 20% | 100,000,000 |

团队代币不得在解锁当天直接进入市场。每轮团队代币进入独立锁仓合约，建议按 24 个月线性释放。

## 十二、创世购买

### 1. 创世套餐

```text
价格：58 TON
总权益：2,800,000 SHORE
立即获得：100,000 SHORE
待解锁：2,700,000 SHORE
```

待解锁部分分 18 轮释放：

```text
每轮释放：150,000 SHORE
```

### 2. 数量限制

```text
总量：1,333 份
单钱包最多购买：10 份
```

### 3. 数学核对

```text
1,333 × 2,800,000 = 3,732,400,000 SHORE
```

```text
立即释放：
1,333 × 100,000 = 133,300,000 SHORE
```

```text
后续释放：
1,333 × 2,700,000 = 3,599,100,000 SHORE
```

```text
每轮购买者释放：
1,333 × 150,000 = 199,950,000 SHORE
```

每轮购买者池为 2 亿 SHORE，剩余 50,000 SHORE 进入官方社区奖励预留钱包。

### 4. 销售规则

创世套餐只在第 0 轮出售。

第 1 轮开启后：

- 永久停止销售；
- 后来的用户不能按 58 TON 补买；
- 后来的用户不能领取过去轮次；
- 未售出的创世权益不能在后期重新按 58 TON 出售；
- 未售权益转入社区奖励预留。

### 5. 金库显示

```text
创世套餐         1 份
立即获得     100,000 SHORE
待解锁     2,700,000 SHORE
已解锁轮次        0/18
```

## 十三、18 轮解锁

### 1. 初始参考价格

```text
$0.00001
```

| 第 18 轮 | $2.62144 | 262,144x |

### 2. 两个公开解锁条件

为了保持用户理解简单，每轮只展示两个条件。

#### 价格条件

SHORE 的 7 日平均价格达到本轮目标。

不得使用某一分钟、某一笔交易或单一低流动性交易池价格触发。

#### 行动条件

平台累计有效任务数量达到本轮目标。

示例：

```text
本轮需要完成任务：500,000 项
当前已经完成：382,600 项
完成度：76.52%
```

两个条件同时满足后，当前轮次进入解锁状态。

### 3. 创世用户领取条件

每份创世套餐在每轮可领取 150,000 SHORE，用户需要满足任一条件：

- 本轮完成 1 项有效任务；
- 本轮获得 300 积分。

未满足条件时：

- 代币不会消失；
- 权益进入待领取状态；
- 用户以后完成任务后仍可领取；
- 项目方不得没收已购买权益。

### 4. 后期加入用户

非创世用户：

- 不能领取以前轮次；
- 不能补买创世套餐；
- 可以完成任务获得积分；
- 可以参与当前和未来轮次积分兑换。

### 5. 轮次数据结构

```ts
type UnlockRound = {
  roundNumber: number;
  targetPriceUsd: string;
  currentSevenDayAveragePriceUsd: string;
  requiredActionCount: number;
  currentActionCount: number;
  priceConditionMet: boolean;
  actionConditionMet: boolean;
  status: "locked" | "ready" | "active" | "completed";
};
```

## 十四、积分兑换 SHORE

### 1. 社区池

每轮提供：

```text
200,000,000 SHORE
```

供积分用户兑换。

### 2. 分配公式

```text
用户获得 SHORE
=
本轮社区池 × 用户有效积分 ÷ 全体用户有效积分
```

### 3. 上限

建议单个用户每轮最多获得社区池的 0.1%。

### 4. 流程

```text
统计本轮有效积分
    ↓
生成用户可领取数量
    ↓
用户连接钱包
    ↓
点击领取
```

## 二十三、TON 智能合约

V1 只开发四个核心合约。

### 1. SHORE Jetton

职责：

- 固定发行 100 亿；
- 禁止额外增发；
- 标准 Jetton 转账；
- 支持销毁；
- 代币元数据不能被单方随意替换。

### 2. Genesis Sale

职责：

- 接收 58 TON；
- 单钱包最多购买 10 份；
- 总量最多 1,333 份；
- 每份立即分配 100,000 SHORE；
- 每份创建 2,700,000 SHORE 锁仓权益；
- 第 1 轮开启后永久关闭购买。

### 3. Vesting Controller

职责：

- 保存用户 18 轮权益；
- 每份每轮 150,000 SHORE；
- 检查轮次是否开启；
- 检查用户是否完成任务或积分条件；
- 支持领取；
- 支持失败重试；
- 防止重复领取。

重要状态：

```text
totalEntitlement
claimedEntitlement
pendingEntitlement
completedRounds
transferNonce
pendingTransfers
```

TON 异步消息处理要求：

- 发起领取前登记 entitlement；
- 发送失败或 Bounce 时不能丢失权益；
- 失败后恢复待领取状态；
- 支持幂等重试；
- 相同 nonce 不得重复支付；
- 重复点击领取不能产生双重发放。

### 4. Community Reward Distributor

职责：

- 保存每轮积分兑换结果；
- 用户连接钱包领取；
- 防止重复领取；
- 未领取奖励按规则保留；
- 每轮奖励结果公开可验证。

## 二十四、反作弊

V1 只检查最重要的风险：

- 同一链接重复提交；
- 同一截图重复提交；
- 多账号使用同一社交主页；
- 同一设备异常批量注册；
- 内容完全相同；
- 无法访问的虚假链接；
- 任务完成速度异常；
- 邀请账号没有完成任何真实任务。

处理方式：

```text
第一次异常：要求重新验证
多次异常：暂停奖励
确认作弊：撤销积分并封禁任务资格
```

不得仅因为用户使用共享网络或公共网络自动封号。

## 三十二、开发边界总结

开发团队必须坚持：

- 不因技术可实现就增加功能；
- 不把后台能力全部暴露给用户；
- 不让用户学习复杂链上术语；
- 不让用户在首页做多项选择；
- 不用复杂任务评分打扰用户；
- 不让钱包成为首次使用门槛；
- 不让分享卡默认暴露真实负债；
- 不自动替用户操作社交账号；
- 不以瞬时价格触发大额代币解锁；
- 不在领取失败时扣除用户权益。

产品判断标准只有一句话：

> 一个第一次打开产品、完全不懂加密货币的人，能否在 3 秒内知道自己要做什么，并在 30 秒内完成第一次核心操作。
